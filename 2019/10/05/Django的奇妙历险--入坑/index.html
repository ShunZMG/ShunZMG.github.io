2 Django简介

2.1 一些说明

这里我假设你已经有了Python 3的基础，不需要太深，了解基本的语法与数据结构就OK，同时，还需要你了解一点关于socket以及wsgi的知识，如果你不还了解，那么你也可以不用担心，我会同步更新简单的socket服务端以及简单的wsgi服务端的相关内容.现在，咱们还是认真的来看Django是如何work的.

2.2 Django简介

  • 好啦，这里咱们就开始说一说咱们的Django.

  • Django基于MTV模式进行开发，后面会介绍，它来源于MVC但又与MVC有不同之处，这个以后我们再进行探讨.

2.3 Django安装

我们首先进入Django的下载页：Django Download Page, Django的下载页介绍了三种方法可以来安装Django

1.获取最新的正式版本，也就是LTS(长期支持)版本，推荐使用pip进行安装，具体版本号参考Django Download

2.获取测试版，也可以使用pip进行安装

3.获取最新的开发版本，使用Git Shell命令来获取

这里我个人推荐第一种方法就好,我相信这也是目前日常开发最常用的.

这里还是要吐槽一下下载速度，不过还好，文件不是很大，经过一段时间的等待，基本上不出问题，Django已经安装完成了，接下来，我们来测试一下Django是否已经安装成功.
进入命令行，输入python，进入python的命令行模式，然后导入django模块，如果没有报错，则django已经安装成功，如果报错，则未安装成功.

3 构建Django项目

3.1 创建项目

首先创建Django项目的目录，在该目录路径下打开终端，输入创建项目命令django-admin startproject [项目名]

等待一会儿，整个项目就会被创建，咱们接着看项目的目录结构.

3.2 Django目录结构

  1. manage.py
    负责执行django中各项操作的文件,如：

    • 启动服务

    • 创建应用(app)

    • 数据库的同步

    • 创建后台超级管理员

      使用方式：python manage.py 或 ./manage.py

  2. 主文件夹（名称与项目名称一致），这里为 MyFirstSite文件夹

    • __init__.py
      项目初始化文件，每当服务器启动时，自动执行

    • settings.py
      项目的主配置文件，包括模板，数据库等配置，其中主要配置简介如下：

      配置项 说明
      BASE_DIR 当前项目所在的绝对路径
      SECRET_KEY 用于生产环境的秘钥
      DEBUG 调试模式，开发过程推荐使用True，上线运行后必须使用False
      ALLOWED_HOSTS 设置允许访问本项目的地址列表，如果没有进行设置，则只有本机，即127.0.0.1/localhost才能访问，推荐使用通配符*，任何表示该机器的地址都可以访问当前项目.如果外部机器想要访问该项目（即从浏览器中访问），可以启用测试服务器，命令为:$ python manage.py runserver 0.0.0.0:8000之后可以在浏览器中输入localhost:8000访问项目
      INSTALLED_APPS 指定已安装的应用(app)，如果创建了自定义应用(app)，则需要在该参数中进行注册，否则，项目无法连接所创建的应用
      MIDDLEWARE
      ROOT_URLCONF 指定根节点的url配置文件
      TEMPLATES 指定模板的相关信息
      DATABASES 指定数据库的相关信息
      LANGUAGE_CODE 指定项目语言，允许修改为zh-Hans (简体中文)
      TIME_ZONE 指定时区，建议修改为 Asia/Shanghai
      STATIC_URL 指定静态文件节点url

5 进行第一次Django请求

5.1 Django的请求流程

  1. Django收到请求时，首先确定根url配置文件，通常根url配置文件的路径由前面提到的ROOT_URLCONF配置项指定，但是如果传入的HttpRequest对象具有urlconf属性，则将其替代ROOT_URLCONF配置.
  2. Django根据获取到的根url配置加载对应的Python模块，并查找变量urlpatterns，通常这是一个列表变量，里面存放的应该是，django.urls.path()（以下使用path()代替）或者django.urls.re_path()（以下使用re_path()代替）的实例.
  3. Django按照urlpatterns中的path()或者re_path()实例的顺序进行模式匹配，并且在第一个与请求的url匹配成功处停止匹配.
  4. 一旦匹配成功，Django就会导入并调用给定的view，该view是一个简单的Python函数（或是基于类的view）,该view得到了下面的一些参数：
    • 一个HttpRequest实例
    • 如果匹配成功的url未返回命名组，那么来自正则表达式的匹配项将作为位置参数提供（其实我也不大懂这句话，但是咱们往下看）
    • 关键字参数由与路径表达式匹配的任何命名部分组成，并且可以由path()或re_path()中的可选参数kwargs
  5. 匹配成功，返回由Python函数处理后的数据，可以使一个字符串，也可以是一个html页面，而如果所有的url都匹配失败，那么Django就会返回一个错误处理页面，如404页面

Django请求的简要流程如下：
浏览器发出请求 –>> urls.py(匹配路由，分发url) –>> views.py(接收到url，进行业务逻辑处理，然后返回数据给浏览器) –>> 浏览器（接受返回的数据并解析显示）
