1 VScode

一些说明

这里介绍的插件都是在Windows 10专业版环境下的Vscode中使用的，在Mac OS中或者Linux中可能有所不同

1.1 个人使用的插件

  • Code Ruuner:基本能满足日常需要的代码运行者
  • One Dark Pro:超爱的深色主题
  • vscode-icons:改善图标
  • open in browser:可以让你在VScode代码编辑区的右键菜单中快速将文件在浏览器中打开，可以在设置中设置默认打开的浏览器，在进行前端页面开发时能带来很大的帮助
  • Bracket Pair Colorizer 2:括号匹配高亮，同一级别的括号用相同的颜色高亮显示，能帮助开发者更好的看清楚括号层次
  • Python:Python开发必备，以及相应的提示进行安装的插件
  • C/C++:C/C++开发必备
  • Java Extension Pack:Java开发必备，这是一个扩展包，会安装一些其他的附加的插件，放心，这些都是必须的，其中包括了Language Support for Java™ by Red Hat、Debugger for Java、Java Test Runner、Maven for Java、Java Dependency Viewer和Visual Studio IntelliCode等插件
  • Peacock:窗口边框美化插件，可以提供随机的窗口边框颜色，也可以自定义哦

1.2 解决使用Code Runner插件运行代码后在OUTPUT窗口下不能输入的问题

  • 解决方案：进入File->Preferences->Settings，在顶部搜索栏搜索Code-runner:Run In Terminal，在出来的选项中将Whether to run code in Integrated Terminal勾上，此时运行代码将在终端(TERNIMAL)中运行，可以进行输入，问题解决

1.3 解决C语言控制台输出中文乱码的问题(同C++解决方案)

  • 情景说明：使用gcc编译器进行编译，利用Code Runner插件进行辅助运行，源代码文件的编码为UTF-8，系统为Windows10，编写好源文件后，用Code Runner转起来，输出的中文乱码，如下：
  • 解决方案：进入VScode界面，进入File->Preferences->Settings，在右边选项中找到Edit in settings.json打开settings.json文件，如果文件中没有code-runner.executorMap该项配置，则添加该项配置，并在配置中找到C语言的运行模板，将其改为cd $dir && gcc $fileName -o $fileNameWithoutExt -fexec-charset=gbk && $dir$fileNameWithoutExt，其实也就是在编译的时候指定了编译器使用的字符集，-fexec-charset=gbk。此时在运行我们的代码，输出问题不再乱码，问题解决

  • Tips:
    • 在settings.json文件中，一般只用输入code-runner.就会弹出候选项，选中executorMap即可自动补全，不过要注意json的格式，比如每一个配置项的名字要使用双引号括起来，后面跟一个冒号，然后是主体配置的内容，每两个配置项之间要使用逗号隔开等
    • 调试C++文件时如果有输出中文乱码情况，同样可以在code-runner.executorMap配置选项中将C++运行模板配置项中加上-fexec-charset=gbk后解决问题

1.4 解决C/C++利用GDB进行调试的问题（Windows）

  • 解决方案：首先要解决MinGW套件安装的问题，有下面两种方案安装，推荐方案二：
    • 方案一：可以去网络上下载MinGW套件进行安装，百度一下教程有很多，需要注意的是安装时最少要安装gcc,gdb等必须工具，但是由于现在墙越来越严实，所以个人不太推荐，这里也就不再详述
    • 方案二：推荐这种方案进行MinGW的安装，我们使用Dev-C++安装后自带的MinGW套件来作为我们VScode中的C/C++编译调试的工具
